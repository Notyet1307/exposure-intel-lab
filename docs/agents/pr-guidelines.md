# Pull Request Guidelines

## Title

```text
<ISSUE-ID>: <imperative summary>
```

Example:

```text
MUL-123: Add tenant-aware audit log export
```

## Required PR sections

- Linked issue
- Summary
- Changes
- Validation
- Screenshots or API examples if relevant
- Risk and rollback
- Security notes

## Merge policy

- Agent-authored PRs require human review.
- High-risk PRs require code owner and security review.
- No direct pushes to main by agents.
- CI must pass unless an owner explicitly documents a non-code failure.
